# Wireshark

В Wireshark 2 типа фильтров:
- Capture Filters (фильтры захвата) - ограничивают запись трафика до захвата
- Display Filters (фильтры отображения) - ограничивают отображаемые пакеты во время захвата

## Примеры Capture Filters

- `host 10.2.2.2` - захват трафика с этим IP
- `port 80` - только трафик порта 80
-  `not port 22` - отображение без трафика порта 22

## Примеры Display Filters

- `ip.addr == 10.2.2.2` - все пакеты от и к ip адресу
- `ip.src == 10.2.2.2` - все пакеты от источника
- `ip.dst == 10.2.2.2` - все пакеты к источнику

### Фильтры протоколов

- `tcp` - Отображение трафика только протокола tcp
- `udp` - Аналогично tcp, только с udp
- `udp.port == 53` - Отображение трафика udp на порту 53
- `icmp` - Отображение icmp-пакетов
- `dns` - Показать весь dns-трафик
- `http` - Показать только http-трафик
- `ftp` - Показать трафик FTP-управления (обычно порт 21).
- `ftp-data` - Показать данные, переданные по FTP.
- `tls` - Показать трафик TLS (зашифрованный HTTP/HTTPS и др.).

Например можно поймать icmp -трафик пинга, для этого нужно зайти в терминал и написать команду ping,
но перед этим в wireshark выбрать сетевой интерфейс и в Display Filters вписать icmp, можно увидеть 
подобные строки

```
468	130.379194457	10.0.2.15	74.125.131.102	ICMP	100	Echo (ping) request  id=0x0003, seq=1/256, ttl=64 (reply in 469)
469	130.417280535	74.125.131.102	10.0.2.15	ICMP	100	Echo (ping) reply    id=0x0003, seq=1/256, ttl=255 (request in 468)
```

- 10.0.2.15 - мой ip
- 74.125.131.102 - ip, на который идет запрос (ping, по протоколу icmp)
